Tentukan apakah tiga segmen garis bisa membentuk segitiga menggunakan `CASE WHEN` dan triangle inequality theorem.

https://leetcode.com/problems/triangle-judgement

## Soal

> Table: `Triangle`
>
> ```
> +-------------+------+
> | Column Name | Type |
> +-------------+------+
> | x           | int  |
> | y           | int  |
> | z           | int  |
> +-------------+------+
> ```
>
> In SQL, (`x`, `y`, `z`) is the primary key column for this table.
>
> Each row of this table contains the lengths of three line segments.

Report for every three line segments whether they can form a triangle.

Return the result table in any order.

The result format is in the following example.

> **Example 1:**
>
> Input:
>
> `Triangle` table:
>
> ```
> +----+----+----+
> | x  | y  | z  |
> +----+----+----+
> | 13 | 15 | 30 |
> | 10 | 20 | 15 |
> +----+----+----+
> ```
>
> Output:
>
> ```
> +----+----+----+----------+
> | x  | y  | z  | triangle |
> +----+----+----+----------+
> | 13 | 15 | 30 | No       |
> | 10 | 20 | 15 | Yes      |
> +----+----+----+----------+
> ```

## Setup Soal

Buat tabel dan insert data contoh:

```sql
CREATE TABLE Triangle (
    x INT,
    y INT,
    z INT,
    PRIMARY KEY (x, y, z)
);

INSERT INTO Triangle (x, y, z) VALUES
    (13, 15, 30),
    (10, 20, 15);
```

## Jawaban SQL

```sql "CASE WHEN"
SELECT
    x, y, z,
    CASE
        WHEN x >= y + z THEN 'No'
        WHEN y >= x + z THEN 'No'
        WHEN z >= x + y THEN 'No'
        ELSE 'Yes'
    END AS triangle
FROM Triangle;
```

## Penjelasan

Soal ini meminta kita untuk menentukan apakah tiga segmen garis dengan panjang `x`, `y`, dan `z` bisa membentuk segitiga.

### Triangle Inequality Theorem

Untuk membentuk segitiga, tiga sisi harus memenuhi **Triangle Inequality Theorem**:

> Jumlah panjang dua sisi segitiga harus **lebih besar** dari sisi ketiga.

Dalam bentuk matematika, untuk membentuk segitiga dengan sisi `x`, `y`, `z`:

```
x + y > z   DAN
x + z > y   DAN
y + z > x
```

Atau secara terbalik (untuk kondisi **TIDAK** membentuk segitiga):

```
x >= y + z  ATAU
y >= x + z  ATAU
z >= x + y
```

Jika salah satu kondisi di atas terpenuhi, maka **TIDAK** bisa membentuk segitiga.

### Breakdown Query

Query ini menggunakan **CASE WHEN** untuk mengecek setiap kondisi:

```sql
CASE
    WHEN x >= y + z THEN 'No'
    WHEN y >= x + z THEN 'No'
    WHEN z >= x + y THEN 'No'
    ELSE 'Yes'
END AS triangle
```

**Cara Kerja CASE WHEN**:

1. Cek kondisi pertama: `x >= y + z`
   - Jika `TRUE`, return `'No'` dan selesai
   - Jika `FALSE`, lanjut ke kondisi berikutnya

2. Cek kondisi kedua: `y >= x + z`
   - Jika `TRUE`, return `'No'` dan selesai
   - Jika `FALSE`, lanjut ke kondisi berikutnya

3. Cek kondisi ketiga: `z >= x + y`
   - Jika `TRUE`, return `'No'` dan selesai
   - Jika `FALSE`, lanjut ke `ELSE`

4. `ELSE 'Yes'`: Jika semua kondisi `FALSE`, berarti bisa membentuk segitiga

### Contoh 1: (13, 15, 30) - TIDAK Bisa Membentuk Segitiga

Mari kita cek:

```
x = 13, y = 15, z = 30

Cek 1: x >= y + z?
       13 >= 15 + 30?
       13 >= 45? → FALSE ✗

Cek 2: y >= x + z?
       15 >= 13 + 30?
       15 >= 43? → FALSE ✗

Cek 3: z >= x + y?
       30 >= 13 + 15?
       30 >= 28? → TRUE ✓
       
Hasil: 'No'
```

Karena `z (30)` lebih besar atau sama dengan `x + y (28)`, maka **TIDAK** bisa membentuk segitiga.

**Visualisasi**:

```
   x=13
   ----
         y=15
         -----
                    z=30
   ----------------------------------

Total x + y = 28, tapi z = 30
Sisi z terlalu panjang! Tidak bisa bertemu untuk membentuk segitiga.
```

### Contoh 2: (10, 20, 15) - Bisa Membentuk Segitiga

Mari kita cek:

```
x = 10, y = 20, z = 15

Cek 1: x >= y + z?
       10 >= 20 + 15?
       10 >= 35? → FALSE ✗

Cek 2: y >= x + z?
       20 >= 10 + 15?
       20 >= 25? → FALSE ✗

Cek 3: z >= x + y?
       15 >= 10 + 20?
       15 >= 30? → FALSE ✗

Semua kondisi FALSE, masuk ke ELSE
Hasil: 'Yes'
```

Semua kondisi `FALSE`, berarti ketiga sisi memenuhi triangle inequality theorem dan **bisa** membentuk segitiga!

**Verifikasi dengan formula positif**:

```
x + y > z?  →  10 + 20 > 15?  →  30 > 15  ✓
x + z > y?  →  10 + 15 > 20?  →  25 > 20  ✓
y + z > x?  →  20 + 15 > 10?  →  35 > 10  ✓

Semua TRUE → Bisa membentuk segitiga!
```

### Hasil Akhir

```
+----+----+----+----------+
| x  | y  | z  | triangle |
+----+----+----+----------+
| 13 | 15 | 30 | No       |
| 10 | 20 | 15 | Yes      |
+----+----+----+----------+
```

### Kenapa Pakai >= Bukan >?

Dalam Triangle Inequality Theorem yang ketat:
- Jumlah dua sisi harus **lebih besar** (bukan sama dengan) sisi ketiga

Jika `x + y = z` (sama dengan), maka ketiga titik akan berada di **satu garis lurus** (collinear), bukan membentuk segitiga.

Contoh:
```
x = 5, y = 5, z = 10

x + y = 10 = z  → Titik berada di garis lurus, bukan segitiga

5---5---10  (garis lurus)
```

Maka kondisi untuk **TIDAK** membentuk segitiga adalah `>=` (lebih besar atau sama dengan).

## Referensi

- `SELECT`: Memilih kolom yang akan ditampilkan.
- `FROM`: Menentukan tabel sumber data.
- `CASE WHEN ... THEN ... ELSE ... END`: Conditional logic untuk mengembalikan nilai berdasarkan kondisi.
- `WHEN`: Mendefinisikan kondisi yang akan dicek.
- `THEN`: Nilai yang dikembalikan jika kondisi TRUE.
- `ELSE`: Nilai default jika semua kondisi FALSE.
- `END`: Menutup statement CASE.
- `AS`: Memberi alias untuk kolom hasil query.
- Triangle Inequality Theorem: Teorema yang menyatakan jumlah dua sisi segitiga harus lebih besar dari sisi ketiga.
- Comparison Operators: `>=` (lebih besar atau sama dengan), `>` (lebih besar dari).
